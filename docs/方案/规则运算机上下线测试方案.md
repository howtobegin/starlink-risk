# 规则运算机上下线测试方案

## 场景一

模拟系统初次上线，先启动计算引擎，然后启动管理平台，上下线多个规则数据。

1. 启动计算引擎时，库中没有规则数据。启动计算引擎后，往数据库中插入'已上线'状态的规则数据。期望得到：运算机正常加载上线，运算机池数据为1.
2. 再往数据库中插入一条'已上线'状态的规则数据。期望得到：运算机正常加载上线，运算机池数据为2.
3. 再往数据库中插入一条'已上线'状态的规则数据。期望得到：运算机正常加载上线，运算机池数据为3.
4. 删除掉数据库中一条'已上线'状态的规则数据。期望得到：运算机正常卸载下线，运算机池数据为2.
5. 删除掉数据库中一条'已上线'状态的规则数据。期望得到：运算机正常卸载下线，运算机池数据为1.
6. 删除掉数据库中一条'已上线'状态的规则数据。期望得到：运算机正常卸载下线，运算机池数据为0.

## 场景二

模拟系统已经上线运行一段时间了，已有多条规则上线运行，但是计算引擎挂掉了，现在需要重新启动计算引擎，需要保证运算机均能正常加载。
后续上下线也正常。

1. 启动计算引擎时，数据库中有2条'已上线'状态的规则数据。期望得到：运算机正常加载上线，运算机池数据为2。
2. 删除掉数据库中一条'已上线'状态的规则数据。期望得到：运算机正常卸载下线，运算机池数据为1.
3. 往数据库中插入一条'已上线'状态的规则数据。期望得到：运算机正常加载上线，运算机池数据为2.
4. 再往数据库中插入一条'已上线'状态的规则数据。期望得到：运算机正常加载上线，运算机池数据为3.
